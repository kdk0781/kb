📖 DSR 계산기 사용 가이드
1
연간 세전 소득 입력
근로소득, 사업소득 등 세전(공제 전) 연간 소득을 입력합니다.
건강보험료 납부 확인서 또는 근로소득 원천징수영수증을 참고하세요.
💡 최소 100만 원 이상 입력해야 계산됩니다
2
대출 종류 선택
보유 또는 신청 예정인 대출 종류를 선택합니다.
· 주택담보 (원리금) — 매월 원금+이자를 함께 상환
· 주택담보 (원금) — 원금균등 상환 방식
· 전세대출 (만기) — 만기 일시 상환, 이자만 DSR 산정
· 오피스텔 — 구입자금이면 '주택담보'로 설정 권장
· 신용대출 — 가상 5년 만기로 DSR 산정
· 카드론 — 가상 3년 만기 고정 (DSR 부담 큼)
3
금리 유형 선택 (주택담보만 해당)
주택담보대출 선택 시 금리 유형(담보)에서 자동 금리 적용이 가능합니다.
· 5년변동 / 5년혼합 — 5년 주기 변동 (스트레스 금리 다름)
· 6,12개월변동 — 코픽스 연동 단기 변동형
· 직접 입력 — 금융기관에서 받은 실제 금리를 직접 입력
💡 금리 유형 선택 시 스트레스 금리가 자동 연동됩니다
4
원금/잔액 및 기간 입력
· 원금/잔액 — 신청 예정 대출금 또는 현재 남은 잔액
· 기간(개월) — 대출 상환 기간 (예: 30년 → 360개월)
· 스트레스 금리는 상품 유형에 따라 자동 설정되며 수동 변경도 가능합니다.
💡 부채 항목 추가 버튼으로 여러 부채를 함께 분석할 수 있습니다
5
DSR 분석 및 한도 확인
모든 정보를 입력한 뒤 DSR 분석 및 한도 확인 버튼을 누르면
· 종합 DSR 지수 (규제선 40% 기준)
· 원금균등 / 원리금균등 방식별 최대 대출 한도
· 현재 추가 가능한 대출 여력
을 확인할 수 있습니다.
6
상환 스케줄 확인 및 리포트 공유
· 전체 상환 스케줄 — 회차별 원금·이자·합계를 N년차 누적 요약과 함께 확인
· DSR 진단 리포트 복사 — 임시 링크(7일 유효)를 생성하여 공유
💡 리포트 링크는 읽기 전용 페이지로 제3자와 공유 가능합니다
7
기존 금리가 변경이 안되있는 경우
· 오른쪽 하단에 있는 새로고침 버튼을 눌러 초기화 시켜주세요
💡 안드로이드는 브라우저 캐시, 아이폰은 사파리 캐시 삭제
⚠️ 주의 — 본 계산기는 참고용 예상 수치입니다.
실제 DSR 산정은 금융기관의 심사 기준에 따라 달라질 수 있으며,
대출 실행 전 반드시 담당 금융기관에 확인하시기 바랍니다.
🛠️ 시스템 관리자 매뉴얼
1
공지사항 강제 노출 (NOTICE_VERSION)
앱 재설치 없이 중요 팝업을 모든 고객에게 다시 띄우는 로직입니다.
· 방법:common.js의 APP_CONFIG.NOTICE_VERSION 값을 올립니다 (예: '0781_7' → '0781_8').
· 원리: "하루 보지 않기" 설정이 구버전 이름(hideNotice_0781_7)으로 저장되어 있으므로, 버전을 올리면 브라우저가 새 공지로 인식하여 강제 노출합니다.
2
실시간 금리 연동 및 UI 자동화
매주 바뀌는 대출 금리를 HTML 수정 없이 즉시 반영합니다.
· 구현: HTML 셀렉트 박스에 숫자가 아닌 value="m5_cycle" 같은 '식별자'를 넣습니다.
· 원리:kb_rates.js 등 외부 파일에서 최신 금리를 주입하면, _syncAllRateSelects() 함수가 화면 UI 글자(예: "5년변동 5.14%")를 자동으로 다시 그리고 계산 로직에도 실시간 반영합니다.
3
PWA 캐시 무효화 (app_main.json 분리)
PWA 웹앱의 강력한 캐시 기능 때문에 업데이트가 지연되는 현상을 차단합니다.
· 구현:dsr.json을 버리고 app_main.json, app_admin.json으로 분리했습니다.
· 원리: 파일명과 시작 주소(start_url)가 달라지면 브라우저가 완전히 새로운 앱으로 인식하므로, 관리자/사용자 충돌 없이 최신 상태로 강제 갱신됩니다.
4
리포트 링크 암호화 및 버전 추적
고객에게 배포되는 진단 리포트 링크의 호환성 유지 및 만료를 제어합니다.
· 구현:v: _C.APP_VERSION(예: 2026.05-C)과 expiry(만료 시간)를 Base64로 암호화하여 URL을 만듭니다.
· 원리: 추후 로직이 대폭 변경되어도 리포트 페이지(report.html)에서 구버전 링크임을 식별해 호환성 처리를 하거나 만료시킬 수 있습니다.
5
리포트 일일 복사량 컨트롤
무분별한 링크 생성(어뷰징)으로 인한 트래픽 낭비를 막습니다.
· 제어:APP_CONFIG.MAX_REPORT_COPIES 에서 하루 허용량(예: 5회)을 설정합니다.
· 원리: 사용자의 로컬스토리지에 오늘 날짜 기준으로 카운팅을 저장하며, 초과 시 알림을 띄우고 버튼 작동을 막습니다.
6
스트레스 DSR 단계 스위치
정부 규제 단계 변경 시 코드 전면 수정 없이 스위치 하나로 대응합니다.
· 제어:APP_CONFIG.STRESS_PHASE 값을 1, 2, 3으로만 변경합니다.
· 원리: 해당 숫자에 따라 가산 금리 반영 비율(25% → 50% → 100%)이 계산 로직 내에서 자동으로 전환 적용됩니다.
⚠️ 유지보수 주의 — 위 시스템 로직들을 수정할 때는 반드시 index.html 상단의 css 하단의 JS 호출 버전(예: ?v=1.2)을 올려주어야 사용자 브라우저 캐시가 갱신됩니다.
📢 업데이트 안내
KB 금리 자동 연동 · 사용 가이드 추가 담보대출 금리가 자동 적용되며, 상단 가이드 버튼에서 사용법을 확인할 수 있습니다.
