📖 DSR 계산기 사용 가이드
1
연간 세전 소득 입력
근로소득, 사업소득 등 세전(공제 전) 연간 소득을 입력합니다.
건강보험료 납부 확인서 또는 근로소득 원천징수영수증을 참고하세요.
💡 최소 100만 원 이상 입력해야 계산됩니다
2
대출 종류 선택
보유 또는 신청 예정인 대출 종류를 선택합니다.
· 주택담보 (원리금) — 매월 원금+이자를 함께 상환
· 주택담보 (원금) — 원금균등 상환 방식
· 전세대출 (만기) — 만기 일시 상환, 이자만 DSR 산정
· 오피스텔 — 구입자금이면 '주택담보'로 설정 권장
· 신용대출 — 가상 5년 만기로 DSR 산정
· 카드론 — 가상 3년 만기 고정 (DSR 부담 큼)
3
금리 유형 선택 (주택담보만 해당)
주택담보대출 선택 시 금리 유형(담보)에서 자동 금리 적용이 가능합니다.
· 5년변동 / 5년혼합 — 5년 주기 변동 (스트레스 금리 다름)
· 6,12개월변동 — 코픽스 연동 단기 변동형
· 직접 입력 — 금융기관에서 받은 실제 금리를 직접 입력
💡 금리 유형 선택 시 스트레스 금리가 자동 연동됩니다
4
원금/잔액 및 기간 입력
· 원금/잔액 — 신청 예정 대출금 또는 현재 남은 잔액
· 기간(개월) — 대출 상환 기간 (예: 30년 → 360개월)
· 스트레스 금리는 상품 유형에 따라 자동 설정되며 수동 변경도 가능합니다.
💡 부채 항목 추가 버튼으로 여러 부채를 함께 분석할 수 있습니다
5
부채 상세 설정순서
1. 대출종류 입력
2. 금리유형 선택시 금리랑 스트레스 금리 자동 셋팅됨
💡 금리 업데이트가 안되는 경우 금리유형 직접입력으로 하여 진행 가능함
6
DSR 분석 및 한도 확인
모든 정보를 입력한 뒤 DSR 분석 및 한도 확인 버튼을 누르면
· 종합 DSR 지수 (규제선 40% 기준)
· 원금균등 / 원리금균등 방식별 최대 대출 한도
· 현재 추가 가능한 대출 여력
을 확인할 수 있습니다.
7
상환 스케줄 확인
· 전체 상환 스케줄 — 원금균등 / 원리금균등 탭을 전환하며 전회차 확인 가능
· 대출별 월 납입액(원리금균등·원금균등 1회차) 나란히 비교 표시
💡 스케줄 표 하단 스크롤로 전 회차를 확인할 수 있습니다
8
DSR 진단 리포트 공유
· 리포트 복사 버튼 클릭 → 상담사 연락처 입력 모달 팝업
   — 전화번호 입력 후 워터마크 추가 : 리포트에 상담사 연락처 포함
   — 닫기(일반 공유) : 워터마크 없이 바로 공유
   — X 버튼 : 공유 취소
· 생성된 링크는 7일간 유효하며, 최초 수신자만 접속 가능
· 문자 공유 시 TinyURL 중간 화면이 나타날 수 있습니다
   → Preview 클릭 또는 10초 후 자동 이동됩니다
💡 리포트 접속 시 "금융 정보 보호 공지" 팝업이 표시됩니다 — 7일 자동 파기 안내
9
기존 금리가 변경이 안되있는 경우
· 오른쪽 하단에 있는 새로고침 버튼을 눌러 초기화 시켜주세요
💡 안드로이드는 브라우저 캐시, 아이폰은 사파리 캐시 삭제
⚠️ 주의 — 본 계산기는 참고용 예상 수치입니다.
실제 DSR 산정은 금융기관의 심사 기준에 따라 달라질 수 있으며,
대출 실행 전 반드시 담당 금융기관에 확인하시기 바랍니다.
🛠️ 시스템 관리자 매뉴얼
1
공지사항 강제 노출 (NOTICE_VERSION)
앱 재설치 없이 중요 팝업을 모든 고객에게 다시 띄우는 로직입니다.
· 수정 위치: common.js → APP_CONFIG.NOTICE_VERSION
· 방법: 값을 한 단계 올립니다 (예: '0781_7' → '0781_8')
· 원리: "다시 보지 않기" 설정이 이전 버전 키(hideNotice_0781_7)로 저장되어 있으므로, 버전이 바뀌면 새 공지로 인식해 강제 노출합니다.
💡 ?v= 버전도 함께 올려야 브라우저 캐시가 즉시 갱신됩니다 (→ 항목 8 참조)
2
실시간 금리 연동 및 UI 자동화
매주 바뀌는 대출 금리를 HTML 수정 없이 자동 반영합니다.
· 금리 원본: kb/interest/script.js 의 r.base · r.stress · r.add.mort 블록
· 로더: kb_rates.js 가 파싱 후 APP_CONFIG.KB_MORTGAGE_RATES · STRESS_RATES 에 주입
· UI 반영: _syncAllRateSelects() 가 셀렉트 옵션 텍스트를 자동 갱신 (value는 식별자 고정)
· 캐시 갱신: kb_rates.js?v= 버전을 올리면 구버전 로컬스토리지 캐시가 자동 무효화됩니다
💡 파싱 실패 시 FALLBACK_RATES 폴백값 자동 적용 — 계산기는 항상 동작합니다
3
PWA 캐시 무효화 (매니페스트 분리)
PWA의 강력한 캐시로 인해 업데이트가 지연되는 현상을 차단합니다.
· 구조: 사용자용 app_main.json / 관리자용 app_admin.json 분리 운용
· 원리: start_url 이 다르면 브라우저가 완전히 다른 PWA 앱으로 인식 → 캐시 충돌 없음
· 강제 갱신: FAB 새로고침 버튼(↻)이 Cache API + Service Worker 전체 초기화 후 재로드
💡 JS·CSS 파일 수정 시에는 항목 8의 ?v= 버전 업이 필수입니다
4
1회성 링크 보안 시스템 (HMAC-SHA256)
고객 배포용 단축 링크의 위변조·재사용을 3단으로 차단합니다.
· ① 만료 시각 검증: 토큰에 ms 타임스탬프(exp)가 포함 — 기간 초과 시 즉시 차단
· ② 서명 검증 (HMAC-SHA256): exp·url·nonce 중 하나라도 조작하면 서명 불일치로 차단
· ③ 1회성 소인: 접속 기기의 localStorage에 nonce 기록 → 같은 기기 재접속 차단

· 서명 키 위치: common.js 와 share.js 각 최상단 _OTL_SIGN_KEY
· ⚠ 중요: 두 파일의 키 값이 반드시 동일해야 합니다 — 키를 바꾸면 기존 링크 전체 즉시 무효화
💡 링크 유효시간 변경은 항목 5에서 별도 설명합니다
5
유효시간 변경 가이드 — 앱 설치 링크 / 리포트 링크
각 링크의 유효시간은 아래 파일의 숫자 하나만 바꾸면 즉시 적용됩니다.

① 앱 설치 / 임시 접속 링크 (share.html)
· 수정 위치: js/admin.js → generateAdminShareLink() 함수 내 SHARE_LINK_TTL_MS
· 동작 방식: 최초 클릭 기기에 바인딩 → 유효기간 내 다회 접속 허용 → 다른 기기에서 열면 차단
· 관리자 기기: 세션이 유효하면 모든 링크 무제한 접속 가능
   —  1 * 60 * 60 * 1000 = 1시간
   —  6 * 60 * 60 * 1000 = 6시간
   —  12 * 60 * 60 * 1000 = 12시간
   —  24 * 60 * 60 * 1000 = 24시간 ← 현재값
   —  48 * 60 * 60 * 1000 = 48시간

② DSR 진단 리포트 링크 (report.html)
· 수정 위치: js/config.js → APP_CONFIG.REPORT_LINK_EXPIRY_DAYS
· 동작 방식: 최초 수신 기기에 바인딩 → 7일간 다회 접속 허용 → 다른 기기에서 열면 차단
· 관리자 기기: 세션이 유효하면 모든 리포트 무제한 접속 가능
   —  REPORT_LINK_EXPIRY_DAYS: 3 = 3일
   —  REPORT_LINK_EXPIRY_DAYS: 7 = 7일 ← 현재값
   —  REPORT_LINK_EXPIRY_DAYS: 14 = 14일
   —  REPORT_LINK_EXPIRY_DAYS: 30 = 30일

③ 공통 주의사항
· 유효시간 변경 후 반드시 ?v= 버전을 올려야 모든 기기에 즉시 반영됩니다
· 이미 발급된 링크의 유효시간은 소급 적용되지 않습니다 (새로 발급한 링크부터 적용)
· 관리자 기기에서 kb_admin_session 이 유효한 동안에는 모든 링크 무제한 접속 💡 테스트 시 관리자 로그인 상태에서는 차단 로직이 우회됩니다 — 일반 시크릿 탭에서 확인하세요
6
리포트 링크 발급량 · 유효기간 제어
고객용 DSR 진단 리포트 링크의 남용 방지 및 만료를 제어합니다.
· 일일 발급 한도: common.js → APP_CONFIG.REPORT_COPY_DAILY_LIMIT (기본 10회)
   — 오늘 날짜 기준 카운팅, 초과 시 버튼 비활성화
· 링크 유효기간: common.js → APP_CONFIG.REPORT_LINK_EXPIRY_DAYS (기본 7일)
   — report.html 에서 만료 여부 자동 판별
· 원리: 로컬스토리지에 날짜별 카운트 저장, 8일 지난 기록은 자동 삭제
💡 리포트 링크는 1회성이 아닌 다회 공유용입니다 — 단축 링크(share.html)와 다릅니다
7
관리자 계정 초기값 설정
최초 배포 전 관리자 아이디·비밀번호·대표 URL을 설정합니다.
· 수정 위치: admin.js 최상단 DEFAULT_CONFIG 블록
   — id: 관리자 아이디
   — pw: 비밀번호 — 배포 전 반드시 변경
   — mainUrl: 로그인 후 이동할 대표 페이지 경로
· 운용 중 변경: 로그인 페이지 하단 ⚙️ 관리자 설정 변경 모달로 즉시 수정 가능
· 저장 위치: localStorage(kb_admin_config) — 기기별 독립 저장
· 세션 유효기간: 로그인 후 24시간, 이후 자동 로그아웃
💡 DEFAULT_CONFIG 는 localStorage 기록이 없을 때만 사용되는 초기값입니다
8
파일 수정 후 캐시 버전 업 (필수)
JS·CSS 파일을 수정할 때마다 호출 버전을 올려야 사용자 브라우저 캐시가 갱신됩니다.
· 수정 위치: index.html 하단 스크립트 태그의 ?v= 쿼리
   — common.css?v=YYYYMMDDHHNN — CSS 수정 시 변경
   — kb_rates.js?v=YYYYMMDDHHNN — 금리 업데이트 시 반드시 변경
   — common.js?v=YYYYMMDDHHNN — 로직 수정 시 반드시 변경
· 권장 값: 수정일시 12자리 (예: 260329 1824 = 26년 03월 29일 18:24)
· kb_rates.js 캐시 키: ?v= 값을 읽어 localStorage 키로 사용 → 버전이 바뀌면 구버전 금리 캐시 자동 폐기
💡 세 파일의 ?v= 값이 달라도 무방합니다 — 각각 독립적으로 캐시를 관리합니다

10
리포트 링크 보안 — 기기 바인딩 & 재공유 차단
동작 방식
   ① 관리자 기기 (kb_admin_session 유효) → 모든 링크 무제한 접속
   ② 최초 수신 기기 → rpt_grant_{nonce} localStorage 바인딩 → 7일간 다회 접속 허용
   ③ 다른 기기에서 링크 복사·전달 → "공유 불가 링크" 에러 표시

앱 설치 링크(share.html)도 동일 구조
   — share_grant_{nonce} 바인딩 → 유효기간 내 다회 접속
   — 관리자 기기에서 발급한 모든 링크 무제한 재접속 가능
💡 테스트 시 시크릿 탭(다른 기기 시뮬레이션)에서 차단 여부를 확인하세요
11
상담사 연락처 워터마크
두 가지 설정 방법
   ① 공유 시 직접 입력 (권장) — 리포트 복사 버튼 클릭 시 팝업에서 입력
      · 입력 시: 리포트에 "CONSULTANT CONTACT + 전화번호" 배너 포함
      · 닫기 클릭: 워터마크 없이 일반 공유
      · X 클릭: 공유 자체 취소
   ② 관리자 설정에 기본값 등록 — admin.html → 설정 변경 → 상담사 연락처 입력
      · 저장 위치: js/admin-login.js DEFAULT_CONFIG.phone
      · 팝업에서 별도 입력 없이도 자동 포함됨
💡 전화번호 형식: 010-0000-0000 (숫자만 입력해도 자동 하이픈 적용)
12
리포트 URL 구조 & 공유 메시지 커스텀
URL 구조 (v3 URL-safe base64)
