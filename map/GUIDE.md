## 1. 디렉토리 구조

```
kdk0781.github.io/
└── map/                        ← 이 앱의 루트 폴더
    ├── index.html              ← 메인 페이지
    ├── manifest.json           ← PWA 설치 정보
    ├── sw.js                   ← 서비스워커 (오프라인 캐시)
    ├── css/
    │   └── common.css          ← 전체 스타일 (다크모드 포함)
    ├── js/
    │   └── app.js              ← 메인 로직
    └── excel/
        └── map.csv             ← ★ 매주 교체하는 시세 데이터
```

> `excel/` 폴더 안의 `map.csv` 파일만 교체하면 데이터가 업데이트됩니다.  
> 아이콘 파일(`map192.png`, `map512.png`)은 상위 `ico/` 폴더에 위치합니다.

## 2. 데이터 업데이트 방법

### 2-1. GitHub 웹에서 직접 업로드 (권장)

1. [GitHub 저장소](https://github.com/kdk0781/kdk0781.github.io) 접속
2. `map/excel/` 폴더 진입
3. 기존 `map.csv` 파일 클릭 → 우측 상단 **연필(✏️) 아이콘** 또는 **삭제(🗑️)**
4. 새 파일 업로드: `Add file` → `Upload files`
5. 파일명을 반드시 `map.csv`로 유지
6. `Commit changes` 버튼 클릭

> ⚠️ **파일명 주의**: 반드시 `map.csv`여야 합니다. 다른 이름 사용 시 앱이 데이터를 찾지 못합니다.

### 2-2. 업로드 후 앱에서 확인

업로드 완료 후 앱에서 **새로고침 버튼**을 눌러 최신 데이터를 받아옵니다.

```
앱 우측 상단 🔄 새로고침 버튼
→ 서비스워커 캐시 삭제 + 페이지 강제 새로고침
→ 최신 CSV 다시 로드
```

GitHub Pages 배포에는 보통 **30초~2분** 정도 소요됩니다.

## 3. CSV 파일 규격

### 3-1. 인코딩

| 항목 | 규격 |
|------|------|
| 인코딩 | **EUC-KR** |
| 줄바꿈 | CRLF 또는 LF 모두 허용 |
| 확장자 | `.csv` |

> UTF-8로 저장 시 한글이 깨져서 표시됩니다. 반드시 EUC-KR로 저장하세요.

### 3-2. 컬럼 구조

앱이 읽는 컬럼 순서 (0-index 기준):

| 컬럼 | 인덱스 | 내용 | 예시 |
|------|--------|------|------|
| A | col[0] | 시/도 | `경기도` |
| B | col[1] | 시/군/구 | `성남시 분당구` |
| C | col[2] | 읍/면/동 | `정자동` |
| D | col[3] | 아파트명 | `파크뷰` |
| E | col[4] | 주택형 (공급면적, suffix 포함) | `84.86T` |
| F | col[5] | 전용면적 | `59.97` |
| G | col[6] | 주택형 (숫자만, suffix 없음) | `84.86` |
| H | col[7] | 구 평형 (주택형 × 0.3025) | `26` |
| I | col[8] | 하한가 (만원) | `"35,000"` |
| J | col[9] | 일반가 (만원) | `"38,000"` |
| K | col[10] | 상한가 (만원) | `"41,000"` |

### 3-3. 기준일 자동 인식

CSV 상단 헤더 영역 (첫 15행 이내)에 아래 형식으로 날짜가 있으면 자동 추출합니다:

```
[2026.03.27~2026.04.02]
```

날짜가 인식되면 앱 화면 우측 하단에 **"기준일 2026.03.27 ~ 2026.04.02"** 형태로 표시됩니다.

### 3-4. 자동 스킵되는 행

다음 키워드가 포함된 행은 자동으로 무시됩니다:

- `전국은행연합회`
- `조견표`
- `절대 수정 금지`
- `대출상담사`
- `시도,시군구`
- `시/도`
- `공급면적`
- `하한가`

### 3-5. 면적 타입 suffix 자동 인식

col[4](주택형)에 숫자 뒤 문자가 붙어 있으면 자동으로 배지로 표시됩니다:

| CSV 값 | 화면 표시 |
|--------|-----------|
| `84.86T` | `테라스` 배지 |
| `84.86P` | `펜트` 배지 |
| `84.86C` | `코너` 배지 |
| `84.86테라스` | `테라스` 배지 (한글 직접 인식) |
| `84.86ABD` | `ABD` 배지 (매핑 없으면 원문) |

## 4. 기능 설명

### 4-1. 검색

```
검색창에 키워드 입력 → 250ms 디바운스 후 자동 필터링
```

- 띄어쓰기로 구분된 **복합 키워드** 지원
  - 예: `중산동 호반` → 중산동이면서 호반이 포함된 단지만 표시
- 검색 대상: **지역(시도+시군구+동)** + **아파트명**
- 대소문자 무시

> **성능 최적화**: 검색 인덱스(`searchKey`)를 CSV 파싱 시 1회만 생성합니다.  
> 검색할 때마다 전체 데이터를 문자열로 재조합하는 방식 대비 훨씬 빠릅니다.

### 4-2. 시/도 지역 칩 필터

헤더 아래 가로 스크롤 가능한 칩 버튼으로 지역을 빠르게 좁힐 수 있습니다.

- CSV에 존재하는 시/도를 자동으로 추출해 칩 생성
- `전체` 칩 선택 시 필터 해제
- 지역 칩 + 검색어를 **동시에** 사용 가능

### 4-3. 정렬

| 옵션 | 설명 |
|------|------|
| 기본순 | CSV 원본 순서 유지 |
| 가나다순 | 아파트명 가나다순 정렬 |
| 가격 낮은순 | 각 단지의 일반가 최저값 기준 오름차순 |
| 가격 높은순 | 각 단지의 일반가 최고값 기준 내림차순 |

### 4-4. ㎡ ↔ 평 단위 토글

헤더 우측의 `[㎡ | 평]` 버튼으로 면적 단위를 즉시 전환합니다.

- **재렌더링 없음**: CSS 클래스(`body.pyeong-mode`) 방식으로 구현
  - ㎡값과 평값을 HTML에 미리 렌더링해두고 CSS로 show/hide만 전환
  - 수만 건 데이터도 토글이 즉각적
- **평형 계산 우선순위**:
  1. CSV `col[7]` 구평형 값이 있으면 그대로 사용 (정수, KB 원본값)
  2. 없으면 `공급면적(㎡) ÷ 3.3058` 자동 계산 (소수점 1자리)
- 전용면적 평형은 항상 `전용면적 ÷ 3.3058`로 계산

### 4-5. 아코디언 카드

각 아파트는 카드 형태로 표시되며, 클릭 시 면적별 시세가 펼쳐집니다.

**카드 헤더에서 보이는 정보:**
- 아파트명 (크게)
- 지역 (시도 + 시군구 + 동)
- 일반가 범위 (예: `35,000 ~ 41,000만`)  ← 열기 전 미리 확인 가능
- 면적 수 (예: `3개`)
- 펼침/접힘 화살표

**카드 내부에서 보이는 정보:**
- 공급면적 / 전용면적 (현재 선택 단위로 표시)
- 면적 타입 배지 (테라스, 펜트 등)
- 하한가 / **일반가** / 상한가 (만원)

> 동일 아파트에서 면적이 여러 개인 경우 모두 펼쳐서 표시됩니다.  
> 한 번에 하나의 아코디언만 열립니다.

### 4-6. 무한 스크롤

한 번에 20개 단지씩 렌더링하며, 스크롤을 내리면 자동으로 추가 로드됩니다.

- `IntersectionObserver` API 사용 (스크롤 이벤트 리스너 없음)
- 하단 300px 전에 미리 로드 (자연스러운 사용 경험)

### 4-7. 강제 새로고침

우측 상단 🔄 버튼:
1. 등록된 서비스워커 전체 해제
2. 브라우저 캐시 전체 삭제
3. localStorage / sessionStorage 초기화
4. 페이지 강제 새로고침 (`reload(true)`)

> 새 CSV를 업로드했는데 앱에 반영이 안 될 때 사용하세요.

### 4-8. 다크모드

`prefers-color-scheme: dark` 미디어 쿼리를 통해 시스템 설정을 자동 감지합니다.  
별도 설정 없이 폰/PC의 다크모드 설정에 따라 자동 전환됩니다.

### 4-9. PWA (홈화면 추가)

모바일 브라우저에서 홈화면에 추가하면 앱처럼 사용할 수 있습니다.

- **iOS Safari**: 공유 버튼 → "홈 화면에 추가"
- **Android Chrome**: 주소창 옆 설치 아이콘 또는 메뉴 → "앱 설치"

설치 후에는 주소창 없이 전체화면으로 실행됩니다.

## 5. 코드 구조 및 동작 원리

### 5-1. 전체 데이터 흐름

```
GitHub에 CSV 업로드
    ↓
앱 실행 시 fetch('excel/map.csv?t=타임스탬프')
    ↓
ArrayBuffer로 수신 → TextDecoder('euc-kr')로 디코드
    ↓
parseCSV() — 행 파싱 → flatData 배열 생성
    ↓
Map으로 그룹화 (키: 시도|시군구|동|아파트명)
    ↓
searchKey 사전 생성 (지역+아파트명 소문자 합산)
    ↓
지역 칩 동적 생성
    ↓
renderInitial() → loadMore() × N회 (무한스크롤)
```

### 5-2. 주요 전역 변수

```javascript
let allGroups      = [];        // 파싱된 전체 그룹 (원본, 불변)
let filteredGroups = [];        // 현재 필터/정렬 적용된 그룹
let activeRegion   = '전체';   // 선택된 시/도 칩
let activeSort     = 'default'; // 정렬 모드
let areaUnit       = 'sqm';     // 면적 단위 ('sqm' | 'pyeong')
let loadedCount    = 0;         // 현재 렌더링된 그룹 수
const LOAD_STEP    = 20;        // 한 번에 렌더링할 그룹 수
```

### 5-3. 그룹 데이터 구조

```javascript
// allGroups 배열의 각 항목
{
    시도: "경기도",
    시군구: "성남시 분당구",
    동: "정자동",
    지역: "경기도 성남시 분당구 정자동",
    아파트: "파크뷰",
    searchKey: "경기도 성남시 분당구 정자동 파크뷰",  // 검색용 소문자
    minPrice: 35000,   // 일반가 최솟값 (정렬/프리뷰용)
    maxPrice: 41000,   // 일반가 최댓값
    rows: [
        {
            공급면적: "84.86",
            전용면적: "59.97",
            공급평형: "26평",
            전용평형: "18.1평",
            suffix: "테라스",   // 빈 문자열이면 배지 미표시
            하한가: "35,000",
            일반가: "38,000",
            상한가: "41,000",
            일반가Raw: 38000,   // 정렬 연산용 숫자값
        },
        // ...
    ]
}
```

### 5-4. 검색 성능 최적화 상세

**기존 방식 (비효율):**
```javascript
// 검색할 때마다 모든 그룹의 모든 행에서 문자열 재조합
const rowsText = group.rows.map(r => Object.values(r).join(' ')).join(' ').toLowerCase();
```

**현재 방식 (최적화):**
```javascript
// 파싱 시 1회만 생성
g.searchKey = `${g.지역} ${g.아파트}`.toLowerCase();

// 검색 시 이미 만들어진 문자열만 확인
result = result.filter(g => terms.every(t => g.searchKey.includes(t)));
```

31,000행 데이터 기준으로 검색 반응 속도가 눈에 띄게 향상됩니다.

### 5-5. ㎡/평 토글 구현 원리

```html
<!-- HTML: 두 단위 모두 렌더링 (보이는 건 CSS가 결정) -->
<span class="area-val u-sqm">84.86㎡</span>    <!-- ㎡ 모드에서 보임 -->
<span class="area-val u-pyeong">26평</span>      <!-- 평 모드에서 보임 -->
```

```css
/* CSS: body 클래스 하나로 전체 제어 */
.u-pyeong { display: none !important; }
.u-sqm    { display: inline !important; }

body.pyeong-mode .u-pyeong { display: inline !important; }
body.pyeong-mode .u-sqm    { display: none !important; }
```

```javascript
// JS: 클래스 토글만 수행 (DOM 재생성 없음)
document.body.classList.toggle('pyeong-mode', areaUnit === 'pyeong');
```

수천 개의 아코디언 행이 렌더링되어 있어도 토글이 즉각 반응하는 이유입니다.

### 5-6. applyFilters() 실행 흐름

```
검색어 입력 / 지역 칩 클릭 / 정렬 변경
    ↓
applyFilters()
    ├── 1. 지역 필터 (activeRegion)
    ├── 2. 키워드 필터 (searchKey.includes)
    └── 3. 정렬 (localeCompare / 숫자 비교)
    ↓
filteredGroups 갱신
    ↓
renderInitial()
    ├── listBody.innerHTML = '' (초기화)
    ├── loadedCount = 0 (리셋)
    ├── 결과 카운트 업데이트
    └── loadMore() 첫 20개 렌더링
```

## 7. 서비스워커 캐시 전략

`sw.js`는 파일 종류에 따라 다른 캐시 전략을 사용합니다.

### Cache-First (정적 자산)

```
요청 → 캐시 확인 → 캐시 있음 → 즉시 반환 (빠름)
                 → 캐시 없음 → 네트워크 요청 → 캐시 저장
```

대상: `index.html`, `common.css`, `app.js`, 아이콘 이미지

### Network-First (CSV 데이터)

```
요청 → 네트워크 요청 → 성공 → 반환 + 캐시 저장 (항상 최신)
                     → 실패 → 캐시에서 반환 (오프라인 대응)
```

대상: `excel/*.csv`

> CSV는 타임스탬프(`?t=타임스탬프`) 쿼리가 붙어 URL이 매번 달라지기 때문에,

## 8. 자주 묻는 질문 (FAQ)

**Q. CSV를 업로드했는데 앱에 반영이 안 돼요.**

A. GitHub Pages 배포에 최대 2분 소요됩니다. 배포 완료 후 앱에서 우측 상단 🔄 **새로고침** 버튼을 눌러주세요. 서비스워커와 브라우저 캐시를 모두 삭제하고 최신 데이터를 받아옵니다.

---

**Q. 기준일이 표시되지 않아요.**

A. CSV 파일 상단 15행 이내에 `2026.03.27` 형식의 날짜가 있어야 합니다. `[2026.03.27~2026.04.02]` 패턴을 권장합니다. 날짜가 없으면 기준일 라벨이 자동으로 숨겨집니다.

---

**Q. 특정 아파트가 검색이 안 돼요.**

A. 검색은 **지역명 + 아파트명** 대상으로만 작동합니다. 시세 금액으로는 검색할 수 없습니다. 아파트명 철자가 CSV와 다르게 입력된 경우도 있으니 확인해보세요.

---

**Q. 면적 단위 토글이 작동하지 않아요.**

A. 앱을 완전히 새로고침 해보세요. 오래된 캐시(구버전 JS)가 남아있는 경우 발생할 수 있습니다. 🔄 새로고침 버튼으로 캐시를 완전히 삭제한 뒤 다시 시도하세요.

---

**Q. 오프라인에서 앱을 쓸 수 있나요?**

A. 한 번이라도 온라인에서 접속했다면, 오프라인에서도 마지막으로 로드했던 데이터로 앱을 사용할 수 있습니다. 단, 새 CSV 로드나 새로고침은 네트워크가 필요합니다.

---

**Q. 아파트가 같은데 여러 행으로 나뉘어 나와요.**

A. 같은 아파트에 면적 타입이 여러 개인 경우 정상입니다. 예: 84㎡ 일반, 84㎡ 테라스, 84㎡ 코너 등이 각각 다른 행으로 표시됩니다. suffix 배지(테라스, 펜트 등)로 구분할 수 있습니다.

## 9. 트러블슈팅

### 데이터가 전혀 표시되지 않을 때

1. **CSV 파일명 확인**: `map/excel/map.csv`로 정확히 존재하는지 GitHub에서 확인
2. **CSV 인코딩 확인**: EUC-KR로 저장되었는지 확인 (UTF-8 금지)
3. **브라우저 개발자 도구 → Console**: 빨간색 오류 메시지 확인
4. **브라우저 개발자 도구 → Network**: `map.csv` 요청의 응답 상태 코드 확인 (200이어야 함)

### 한글이 깨져서 보일 때

CSV 파일이 UTF-8로 저장된 경우입니다.  
Excel에서 저장 시 "CSV (MS-DOS)" 또는 "CSV (Macintosh)" 형식을 선택하거나,  
메모장에서 "다른 이름으로 저장 → 인코딩: ANSI"로 저장하세요.

### 가격이 '-'로 표시될 때

CSV에서 해당 행의 col[8]~col[10] 값이 `0` 이거나 비어있는 경우입니다.  
원본 CSV 데이터를 확인해주세요.

### 평형이 이상하게 계산될 때

CSV `col[7]`(구평형) 값이 있으면 그 값을 우선 사용합니다.  
col[7]이 비어있거나 `0`이면 `공급면적 ÷ 3.3058`로 자동 계산합니다.  
KB 원본 CSV에는 col[7]에 구평형이 포함되어 있으므로 정상적으로 표시됩니다.

### 서비스워커 관련 문제 (캐시가 너무 오래됨)

브라우저 개발자 도구에서 직접 초기화할 수 있습니다:

```
개발자 도구 → Application → Service Workers → Unregister
개발자 도구 → Application → Storage → Clear site data
```

또는 앱 내 🔄 **새로고침** 버튼이 동일한 동작을 자동으로 수행합니다.
